# Internationalization (i18n) — v1.0

## Supported Locales
- he-IL (default, RTL)
- en-US (LTR)

---

## Rules
- No hardcoded strings in UI
- All labels via translation keys
- Hebrew UI is primary

---

## Database i18n Strategy
- Codes/keys stored in DB
- Optional bilingual fields:
  - asset_types.name_he / name_en
  - asset_property_definitions.label_he / label_en
  - ticket_status_definitions.name_he / name_en
- If DB label missing → fallback to UI dictionary

---

## Example Translation Keys
ticket.status.new
ticket.status.in_progress
ticket.status.closed

asset.type.nvr
asset.type.router

worklog.type.onsite
worklog.type.remote

---

## RTL Requirements
- `<html dir="rtl" lang="he">`
- Icons mirrored where needed
- Tables right-aligned by default

---

## User Preference
- Each user has preferred_locale
- UI auto-switches
- Manual switch allowed

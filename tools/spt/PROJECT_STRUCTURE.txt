KEY FILES EXPLAINED
-------------------

🚀 ENTRY POINTS
  • app/main.py              - Start here! Main CLI application
  • test_text_only.py        - Quick testing without audio files

🧩 CORE SERVICES
  • speech_to_text.py        - Converts audio → text + language
  • translate.py             - Translates between ru/he/en
  • normalize_ticket.py      - Casual speech → professional ticket

⚙️ CONFIGURATION
  • .env                     - Your secrets (API key, settings)
  • .env.example             - Template for .env
  • requirements.txt         - pip install -r requirements.txt

📚 DOCUMENTATION
  • README.md                - Complete guide (read first!)
  • QUICKSTART.md            - Get running in 5 minutes
  • ARCHITECTURE.md          - Deep dive into design

  • normalize_ru.txt         - Russian ticket formatting rules
  • normalize_he.txt         - Hebrew ticket formatting rules


EXECUTION FLOW
--------------

1. User runs:     python app/main.py audio.mp3

2. main.py loads: .env configuration

3. Pipeline runs:
   ├─> speech_to_text.py      (audio → text + lang)
   ├─> translate.py           (if needed)
   └─> normalize_ticket.py    (text → ticket)

4. Output:        Formatted results to console

ISOLATION FROM CRM
------------------

This is a SANDBOX project:
  ✅ Self-contained in tools/spt/
  ✅ No imports from main CRM code
  ✅ No database connections
  ✅ No shared dependencies
  ✅ Independent virtual environment

You can:
  • Modify SPT without affecting CRM
  • Delete SPT without breaking CRM
  • Run SPT while CRM is running
  • Copy SPT to another project


NEXT STEPS
